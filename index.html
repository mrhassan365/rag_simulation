RAG in simple words

Read this once before using the simulator.

What RAG is

RAG means the system first looks up relevant information from external knowledge, then gives that information to the LLM as context before the LLM answers.

What RAG is not

RAG is not the LLM magically “knowing” your PDF. The LLM does not directly read the whole knowledge base on every question. The system retrieves only the most relevant chunks first.

The easiest mental model

Think of RAG like an open-book test:

  • Without RAG = answer from memory
  • With RAG = first open the right pages, then answer

Why it matters

RAG helps answers become more specific, more grounded, and more tied to your actual documents instead of vague general knowledge.

What you should notice

The simulator is most useful when you watch for these differences.

Without RAG

  • more generic
  • more cautious or vague
  • less tied to your actual source docs
  • can sound reasonable without being exact

With RAG

  • more exact
  • grounded in retrieved evidence
  • uses specific details from source docs
  • lets you inspect which chunks supported the answer
